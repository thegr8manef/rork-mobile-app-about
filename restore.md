## How it runs

- **Automatically** ŌĆö `expo prebuild` triggers `withPrebuildDiff.js` which runs restore ŌåÆ diff
- **Manually** ŌĆö `npm run prebuild:restore`

---

## Workflow Guide ŌĆö `withPrebuildDiff.js` plugin

### What is `withPrebuildDiff.js`?

An Expo config plugin registered in `app.json`. It hooks into the end of every
`expo prebuild` and automatically runs two steps in order:

1. `restore-android.js` ŌĆö copies customized files back from
   `PreviousAndroid/` and `PreviousIos/`
2. `prebuild-diff.js` ŌĆö checks for any remaining differences and reports them

You never need to run restore or diff manually after a clean prebuild.

---

### The only command you need

```bash
npx expo prebuild --clean
```

Internally this does:

```text
1. Expo wipes android/ and ios/ folders
2. Expo generates fresh native files
3. withPrebuildDiff.js runs automatically at the end:
     ŌåÆ restore-android.js  (restores customized files ŌĆö only if marker is missing)
     ŌåÆ prebuild-diff.js    (reports any remaining differences)
4. android/ and ios/ are ready to build
```

Then build:

```bash
npx expo run:android
```

---

### How the restore knows when to run

The restore checks for a "marker" file that Expo **never** generates:

| Platform | Marker file                                               |
| -------- | --------------------------------------------------------- |
| Android  | `android/app/src/main/java/.../BuildConfigModule.kt`      |
| iOS      | `ios/SecureSignModule.swift`                              |

- Marker **absent** ŌåÆ expo just wiped the folder ŌåÆ restore runs
- Marker **present** ŌåÆ restore already ran ŌåÆ skip (nothing to do)

This means running `expo prebuild --clean` twice in a row is safe ŌĆö the second
time the restore is skipped automatically.

---

### Individual commands (manual use only)

| Command | What it does |
| ------- | ------------ |
| `npx expo prebuild --clean` | Full flow: generate + auto-restore + auto-diff |
| `npm run prebuild:restore` | Restore only (skips if already done) |
| `npm run prebuild:diff` | Diff check only, no prebuild, no restore |
| `npm run prebuild:check` | `expo prebuild` (no `--clean`) + diff only |

> `prebuild:check` is useful for a quick diff check without wiping the native folders.
